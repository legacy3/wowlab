# Rotation Editor - UX Concept & Integration Plan

> A comprehensive guide for the rotation editor user flow, navigation, and integration with the simulation engine.

## Executive Summary

The rotation editor enables users to **create**, **test**, and **share** custom rotation scripts that define ability priorities for WoW class/spec combinations. The editor should feel like a lightweight IDE with immediate feedback loops - similar to CodePen or JSFiddle, but specialized for rotation development.

## Script Format & Validation

### Proposed Script Format (SimC-compatible)

```simc
# Beast Mastery Hunter - Single Target
# Author: darkmage
# Updated: 2024-01-15

actions=auto_attack

# Priority: Kill Command > Barbed Shot > Cobra Shot
actions+=/kill_command
actions+=/barbed_shot,if=pet.main.buff.frenzy.remains<gcd
actions+=/bestial_wrath
actions+=/aspect_of_the_wild
actions+=/cobra_shot,if=focus>50

# Cooldowns
actions+=/bloodshed
actions+=/call_of_the_wild
```

### Validation Rules

1. **Syntax validation** - Check for valid action list format
2. **Spell existence** - Verify spell names/IDs exist for the class/spec
3. **Conditional syntax** - Validate `if=` expressions
4. **Warnings** - Missing common abilities, suboptimal priority

## Technical Notes

### Existing Hooks to Leverage
- `useRotation(id)` - Fetch single rotation
- `useRotationMutations()` - Create/update/delete
- `useSimulation()` - Run simulation with job tracking
- `useList<Rotation>()` - List rotations with filters

### Key Atoms
- `fightDurationAtom` - Simulation duration
- `iterationsAtom` - Number of iterations
- `targetTypeAtom` - Target type (patchwerk, movement, etc.)
- `computingDrawerOpenAtom` - Controls simulation drawer

### Simulation Flow
1. Editor calls `run(rotation, duration)`
2. Creates job in `simJobsAtom`
3. Opens computing drawer
4. Loads spells/auras via data provider
5. Creates Effect runtime
6. Runs simulation loop with progress updates
7. Completes and shows results
