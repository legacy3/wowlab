# WoW Lab <Badge>v0.1.0</Badge>

<Alert title="Work in Progress">
  This is an early release. Things will break and features are incomplete.
</Alert>

WoW Lab is a combat simulation platform for World of Warcraft. Build rotations in a visual editor, run distributed simulations on your own hardware, and query game data programmatically.

## Features

<CardGrid columns={3}>
  <FeatureCard title="Build Rotations">
    Visual editor for spell priorities and conditions. Serialize to JSON and
    save to the database.
  </FeatureCard>
  <FeatureCard title="Run Simulations">
    Distributed on your hardware. Rust engine with JIT compilation for maximum
    performance.
  </FeatureCard>
  <FeatureCard title="Query Game Data">
    MCP server exposes spell/item data to AI tools, scripts, and applications.
  </FeatureCard>
</CardGrid>

## How it works

<Mermaid
  chart={`
flowchart LR
    Editor[Rotation Editor] --> DB[(Database)]
    DB --> Node[Local Node]
    Node --> Engine[Rust Engine]
    Engine --> Results[Results]
    Results --> Portal[Portal]
`}
/>

The portal imports the engine as WASM for type sharing and rotation validation, but actual simulation runs on your local node where JIT compilation is possible.

## Getting Started

<Steps>
  <Step step={1} title="Create a rotation">
    Open the [rotation editor](/rotations/editor), select your spec, add actions
    and conditions, then save.
  </Step>
  <Step step={2} title="Start a local node">
    Install and run a local node on your machine, Docker, or cloud to receive
    simulation jobs.
  </Step>
  <Step step={3} title="Run simulations">
    Go to the [simulate page](/simulate), select your rotation, configure the
    fight, and run.
  </Step>
  <Step step={4} title="View results">
    Results stream back to the portal in real-time via Supabase Realtime.
  </Step>
</Steps>

## Query WoW Data

The [MCP Server](/dev/docs/reference/07-mcp-server) exposes WoW spell/item data to AI assistants and other tools.

<Tabs defaultValue="claude">
  <TabList>
    <TabTrigger value="claude">Claude</TabTrigger>
    <TabTrigger value="cursor">Cursor</TabTrigger>
    <TabTrigger value="api">Direct API</TabTrigger>
  </TabList>
  <TabContent value="claude">
    Configure the MCP server in Claude Desktop to query spells, items, and game
    mechanics directly in conversations.
  </TabContent>
  <TabContent value="cursor">
    Add the MCP server to Cursor for inline spell lookups while writing rotation
    code or game addons.
  </TabContent>
  <TabContent value="api">
    Call the MCP server directly from scripts or applications for batch spell
    data queries.
  </TabContent>
</Tabs>
