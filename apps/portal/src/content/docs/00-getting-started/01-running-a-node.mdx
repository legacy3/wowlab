# Running a Node

Simulations run on local nodes, not in the browser. The Rust engine uses JIT compilation which browsers cannot support.

## Quick start

1. Download the node app from [GitHub Releases](https://github.com/legacy3/wowlab/releases)
2. Run the app - you'll see a claim code like `XXXX-XXXX`
3. Go to [wowlab.gg/nodes](https://wowlab.gg/nodes) and enter the code
4. The node links to your account and starts accepting jobs

Two versions are available:

| Version | Use case |
| ------- | -------- |
| `node-gui` | Desktop app with UI, stats, settings |
| `node-headless` | Servers, containers, headless systems |

## Why local nodes?

Browsers cannot JIT-compile arbitrary code. The simulation engine compiles rotations to native machine code using Cranelift for maximum performance. This requires running outside the browser sandbox.

Benefits:

- Your hardware runs the math
- No server costs
- Full CPU utilization
- Privacy (simulation data stays local)

## Configuration

### CPU cores

The GUI app has a slider to control how many CPU cores the node uses. By default, it uses the optimal number for your system:

- Apple Silicon: Performance cores only (excludes efficiency cores)
- x86: Physical cores only (excludes hyperthreads)

### Logging

Control log verbosity with the `RUST_LOG` environment variable:

```bash
RUST_LOG=debug ./node-headless
RUST_LOG=node_core=trace ./node-gui
```

### Config file location

The node stores its configuration (node ID, API credentials) in:

| Platform | Path |
| -------- | ---- |
| macOS | `~/Library/Application Support/gg.wowlab.wowlab-node/config.ini` |
| Linux | `~/.config/wowlab-node/config.ini` |
| Windows | `%APPDATA%\wowlab\wowlab-node\config.ini` |

## Docker

Run the headless node in a container:

```bash
docker run -d ghcr.io/legacy3/wowlab-node
```

Mount volumes to persist config and logs:

```bash
docker run -d \
  -v ~/.config/wowlab-node:/root/.config/wowlab-node \
  -v ~/.local/share/wowlab-node/logs:/root/.local/share/wowlab-node/logs \
  ghcr.io/legacy3/wowlab-node
```

## Building from source

Clone and build the node:

```bash
git clone https://github.com/legacy3/wowlab.git
cd wowlab

# Desktop app
cargo build --release -p node-gui
./target/release/node-gui

# Headless version
cargo build --release -p node-headless
./target/release/node-headless
```

## CLI options

Both binaries support these options:

```
node-gui [OPTIONS] [COMMAND]
node-headless [OPTIONS] [COMMAND]

Options:
  --no-update    Skip automatic update check

Commands:
  run            Start the node (default)
  update         Check and install updates
  update --check Check for updates without installing
```

## Auto-updates

The node checks for updates on startup. The headless version auto-updates and restarts. The GUI version shows a prompt.

Skip the update check with `--no-update`:

```bash
./node-headless --no-update
```
