# Content Style Guide

Guidelines for writing MDX content in the docs and blog.

## Writing Style

- **Be concise.** Short sentences. No filler.
- **Be accurate.** Verify claims against code. No speculation.
- **Be direct.** Tell the reader what to do.
- **Use present tense.** "The engine compiles" not "The engine will compile".

## When to Use Each Element

### Native Markdown (prefer these)

| Element         | When to Use                                     |
| --------------- | ----------------------------------------------- |
| `## Heading`    | Major sections. H2 for main, H3 for subsections |
| `**bold**`      | Key terms on first use                          |
| `` `code` ``    | Function names, variables, file paths           |
| `- list`        | 3+ related items                                |
| `[link](/path)` | Navigation, references                          |
| Tables          | Comparing 3+ items with shared attributes       |
| Code blocks     | Examples (always include language)              |

### Custom Components

| Component       | When to Use                         | When NOT to Use    |
| --------------- | ----------------------------------- | ------------------ |
| `<Alert>`       | Warnings, breaking changes, gotchas | General info       |
| `<Badge>`       | Version numbers, status labels      | Decoration         |
| `<Card>`        | Grouping related info with title    | Single paragraphs  |
| `<CardGrid>`    | 2-3 feature cards side by side      | More than 3 cards  |
| `<Collapsible>` | Optional deep-dive content          | Required reading   |
| `<Kbd>`         | Keyboard shortcuts only             | Code or commands   |
| `<Mermaid>`     | Process flows, state diagrams       | Simple lists       |
| `<Steps>`       | Multi-step procedures               | Single actions     |
| `<Tabs>`        | Alternative implementations         | Sequential content |

## Component Reference

See [MDX Components](/dev/docs/guides/02-mdx-components) for syntax and examples.

## Code Blocks

Always specify language:

```rust
fn main() {}
```

Supported: `rust`, `typescript`, `json`, `bash`, `yaml`, `jsx`

## Links

- Internal: `[text](/dev/docs/section/page)`
- External: `[text](https://example.com)` - opens in new tab
- Anchor: `[text](#heading)` - slugified heading

## Do NOT

- Add placeholder content ("TODO", "Coming soon")
- Make claims without verifying against code
- Use components for decoration
- Nest components excessively
- Write walls of text without structure

## File Naming

- Docs: `{order}-{slug}.mdx` (e.g., `00-overview.mdx`)
- Blog: `{date}-{slug}.mdx` (e.g., `2025-12-hello.mdx`)
